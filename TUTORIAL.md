# TV Remote 使用教程

远程控制雷鸟电视（Android TV），通过网页直接点击电视实时画面操控。

---

## 一、电视端安装

### 1. 开启 ADB 调试（一次性设置）

遥控器操作：

```
设置 → 关于 → 软件版本（连续按确认键 7 次）→ 开启开发者模式
设置 → 开发者选项 → USB 调试 → 开启
```

> 如果电视上没有「开发者选项」，先到「设置 → 关于」里找到「内部版本号」或「软件版本」，连续按 OK 键 7 次即可解锁。

### 2. 安装 APK

**方式 A：用 U 盘安装（推荐）**

1. 在电脑上，用 Android Studio 打开 `android/` 目录
2. 菜单栏：`Build → Build Bundle(s) / APK(s) → Build APK(s)`
3. 构建完成后，APK 在 `android/app/build/outputs/apk/debug/app-debug.apk`
4. 把 APK 复制到 U 盘，插到电视上
5. 电视上用文件管理器打开 U 盘，点击 APK 安装

**方式 B：用 ADB 安装**

电视和电脑连在同一个 WiFi 下：

```bash
# 电脑端执行
adb connect <电视IP地址>:5555
adb install app-debug.apk
```

### 3. 配置 App

1. 在电视上打开 **TV Remote** 应用
2. 输入一个 **设备ID**（自己随便起，比如 `livingroom`、`bedroom-tv`，英文+数字）
3. Broker 地址保持默认 `tcp://broker.emqx.io:1883` 不用改
4. 点击 **启动服务**
5. 会弹出「忽略电池优化」提示，选择 **允许**（防止系统杀后台）
6. 通知栏出现「TV Remote 已连接」即表示成功

> 配置会自动保存，下次开机自动启动服务，无需再操作。

## 二、手机/电脑控制端

### 打开控制页面

**手机端**（推荐）：

1. 用手机浏览器打开：
   ```
   https://<你的部署地址>/index.html
   ```
   或者本地直接打开文件：把 `web/index.html` 下载到手机，用浏览器打开

2. 输入和电视端**相同的设备ID**
3. 点击 **连接**
4. 等待几秒，看到电视实时画面和 **LIVE** 绿色徽章，连接成功

### 本地测试

在电脑上直接用浏览器打开文件即可：

```bash
open web/index.html   # macOS
start web/index.html  # Windows
```

或者启动一个本地服务器（解决跨域问题）：

```bash
cd web && python3 -m http.server 8080
# 然后手机浏览器访问 http://<电脑IP>:8080
```

## 三、使用方法

### 核心操作：点击画面

```
┌─────────────────────┐
│                     │
│   电视实时画面        │  ← 直接点击画面上的 App 图标、按钮、菜单
│   👆 点哪 = 摸哪     │     就像手指直接触摸电视屏幕一样
│                     │
│     LIVE  6 fps     │
│   点击画面 = 遥控触摸  │
└─────────────────────┘
  [◁ 返回]  [⌂ 主页]  [⏻ 电源]
```

| 操作 | 方法 |
|------|------|
| 点击电视上的按钮 | 直接在画面上点那个按钮 |
| 滑动选择 | 点击画面上的方向箭头或列表项 |
| 打开 App | 点击画面上的 App 图标 |
| 输入文字 | 点击电视键盘上的字母 |
| 返回上一级 | 点击画面上的返回按钮，或点底部 **◁ 返回** |
| 回到桌面 | 点底部 **⌂ 主页** |

### 底部硬件按键

| 按键 | 功能 | 使用场景 |
|------|------|---------|
| ◁ 返回 | 返回上一级 | 画面上找不到返回按钮时 |
| ⌂ 主页 | 回到桌面 | 快速退出当前 App |
| ⏏ 电源 | 电源键 | 锁屏/唤醒 |

## 四、连接状态说明

| 指示灯 | 状态 | 含义 |
|--------|------|------|
| 🟢 绿色 | 已连接 | TV 在线，可以控制 |
| 🔴 红色 | 未连接 | TV 不在线或未启动服务 |
| ⚫ 灰色 | 未连接 | 还没输入设备ID |

| 画面标记 | 含义 |
|----------|------|
| LIVE 绿色 | 正在接收实时画面，可以点击控制 |
| 等待画面... | 已连接但还没收到画面，等几秒 |
| 未连接 | 没连上，检查设备ID和TV状态 |

## 五、常见问题

### Q: TV 服务总是被杀？

1. 电视设置 → 应用 → TV Remote → 电池 → 设为「不限制」
2. 电视设置 → 应用 → 特殊权限 → 后台运行 → 允许 TV Remote

### Q: 画面延迟大 / FPS 低？

- MQTT 免费 Broker 有带宽限制，画面质量已自动压缩（640px，约 6fps）
- 家庭内网效果更好
- 如果卡顿严重，检查 WiFi 信号

### Q: 点击位置不准？

- TV 会自动上报真实分辨率，网页端自动换算
- 如果偏差较大，可能是电视分辨率不在标准范围，可联系调整

### Q: 如何更换设备ID？

1. 电视上打开 TV Remote App
2. 输入新设备ID → 点启动服务
3. 手机上输入相同新设备ID → 点连接

### Q: 卸载 App 会残留文件吗？

不会。App 不写任何外部文件，卸载后完全清除。

### Q: 安全吗？

- 设备ID 作为简易凭证，只有知道设备ID的人才能控制
- MQTT 连接走公网 Broker，数据传输经过 TLS（WebSocket 端）
- 建议设备ID 设置复杂一些，不要用简单词汇
